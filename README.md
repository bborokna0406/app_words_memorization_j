# 일본어 단어 암기 앱

## GitHub Pages에 처음 배포하기

1. 이 폴더 안의 파일과 `assets` 폴더를 GitHub 저장소의 최상위에 올립니다.
2. GitHub 저장소의 **Settings -> Pages**에서 배포할 브랜치와 루트 폴더를 선택합니다.
3. 생성된 HTTPS 주소를 휴대폰 브라우저에서 엽니다.
4. 브라우저 메뉴에서 **홈 화면에 추가**를 선택합니다.

## 앱 업데이트하기

컴퓨터에서 파일을 수정한 뒤 같은 GitHub 저장소에 커밋하고 푸시합니다. 휴대폰 앱이 인터넷에 연결된 상태로 열리면 새 버전을 확인하며, 업데이트가 발견되면 자동으로 다시 불러옵니다.

## 저장된 단어 유지

앱 파일 업데이트와 단어 데이터는 서로 분리되어 있습니다. 같은 GitHub Pages 주소를 계속 사용하고 `app.js`의 `STORAGE_KEY`를 바꾸지 않으면 앱을 업데이트해도 저장된 단어가 유지됩니다.

다만 다음 경우에는 휴대폰의 단어가 사라질 수 있습니다.

- 브라우저의 사이트 데이터나 캐시를 직접 삭제한 경우
- 홈 화면 앱을 삭제하면서 사이트 데이터도 삭제한 경우
- GitHub 저장소 또는 Pages 주소를 변경한 경우
- 다른 휴대폰이나 다른 브라우저에서 앱을 연 경우

중요한 단어는 앱 오른쪽 위의 백업 버튼으로 JSON 파일을 주기적으로 보관하세요. 코드 업데이트는 자동으로 반영되지만, 여러 기기 사이의 단어 데이터가 자동 동기화되는 것은 아닙니다.

## 업데이트할 때 버전 올리기

앱 파일을 수정해 배포할 때 `app.js`의 `APP_VERSION`과 `version.json`의 `version` 값을 같은 새 번호로 함께 변경하세요. 예: `2026.06.23.1` -> `2026.06.23.2`. 이 번호가 달라지면 홈 화면 앱이 새 배포를 감지하고 자동으로 다시 불러옵니다.

버전 번호를 바꿔도 `STORAGE_KEY`는 변경하지 마세요. `STORAGE_KEY`가 유지되어야 휴대폰에 저장된 단어가 계속 남습니다.
